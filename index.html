How Strassen Works

Overview

1 The Intuitive Idea

Given two n×n matrices A and B, we want to compute their product C = A×B. Standard multiplication requires n³ scalar multiplications, which grows very fast.

In 1969, Volker Strassen found a way to reduce the number of recursive multiplications from 8 to 7 at each level. Because multiplications are the most expensive operation (especially for large sub-matrices), this small reduction compounds recursively and lowers the overall complexity.

2 Divide and Conquer

Strassen’s algorithm follows the divide-and-conquer paradigm:

  • Divide: Split each n×n matrix into 4 quadrants of size (n/2)×(n/2):
    A = [a  b]     B = [e  f]
        [c  d]         [g  h]
  • Conquer: Compute 7 intermediate products (P1–P7) by cleverly combining quadrants, then recursively multiply each pair.
  • Combine: Assemble the four result quadrants C11, C12, C21, C22 using only additions and subtractions of P1–P7.

3 The 7 Strassen Products (P1–P7)

Each product mixes quadrants from A and B so their results can later be combined into all four quadrants of C:

P1 = (a + d) · (e + h)
P2 = (c + d) · e
P3 = a · (f − h)
P4 = d · (g − e)
P5 = (a + b) · h
P6 = (c − a) · (e + f)
P7 = (b − d) · (g + h)

The result quadrants are then assembled using only additions and subtractions:

C11 = P1 + P4 − P5 + P7
C12 = P3 + P5
C21 = P2 + P4
C22 = P1 − P2 + P3 + P6

The key insight is that each product contributes to multiple result quadrants simultaneously, which is why only 7 multiplications are needed instead of 8.

4 Complexity

Strassen’s recurrence and its solution by the Master Theorem:

T(n) = 7 · T(n/2) + O(n²)  ⇒  O(nlog27) ≈ O(n2.807)

With a=7 sub-problems of size n/2 and O(n²) combine work, log27 ≈ 2.807 becomes the exponent. The additions at each level are asymptotically dominated by the recursive term.

5 Why 3×3 Matrices Cannot Be Used Directly

Strassen requires splitting into 4 equal quadrants of size (n/2)×(n/2). This only works when n is even. For 3×3:

  • n/2 = 1.5 — the matrix cannot be evenly split into 4 quadrants.
  • The recursive structure breaks because sub-matrices would have unequal sizes.

Solution: Pad with zeros to the next power of 2.

Original 3×3:
[a  b  c]      Padded to 4×4:
[d  e  f]   →  [a  b  c  0]
[g  h  i]      [d  e  f  0]
                [g  h  i  0]
                [0  0  0  0]

After computing the 4×4 result, the padded row and column are discarded, leaving the correct 3×3 answer. Multiplying by zero does not affect the original entries.

6 When Is Strassen Useful?

  • Large dense matrices (n > 64): The asymptotic advantage becomes measurable and grows with n.
  • Small matrices: The extra additions and recursive overhead can make Strassen slower. Most implementations switch to a simpler method below a crossover threshold.
  • Parallel computing: The 7 sub-problems are independent and can run simultaneously on multiple cores.

How Naive Multiplication Works

Overview

1 The Intuitive Idea

Given two n×n matrices A and B, we want to compute their product C = A×B. The naive (standard) algorithm computes each entry of C one at a time by taking a dot product: multiply corresponding elements from a row of A and a column of B, then sum the results.

This is the most straightforward way to multiply matrices — it directly follows the mathematical definition.

2 Step-by-Step Process

The algorithm uses three nested loops controlled by indices i, j, and k:

  • i (outer loop) — selects which row of C we are computing.
  • j (middle loop) — selects which column of C (i.e., which cell C[i][j]).
  • k (inner loop) — walks along row i of A and column j of B simultaneously, multiplying A[i][k] × B[k][j] and accumulating the sum.

For each result cell C[i][j], the inner loop performs n multiplications and n−1 additions, building up the dot product one term at a time:

For C[i][j]:
sum = 0
sum += A[i][1] · B[1][j]   (k=1)
sum += A[i][2] · B[2][j]   (k=2)
  ⋮
sum += A[i][n] · B[n][j]   (k=n)
C[i][j] = sum

This process repeats for every combination of i and j, filling the entire result matrix.

3 The Formula

Each entry of the result matrix is defined as:

C[i][j] = ∑k=1n A[i][k] · B[k][j]

In words: to get C[i][j], take row i of A and column j of B, multiply them element-by-element, and add up all the products. This is called the dot product of that row and column.

4 Complexity

There are n² cells in C, and each cell requires a loop of n multiplications:

n² cells × n multiplications per cell = n³ total multiplications  ⇒  O(n³)

The number of additions is n²(n−1), which is also O(n³). The overall time complexity is O(n³).

5 When Is Naive Multiplication Useful?

  • Small matrices: The algorithm has low overhead and is simple to implement, making it the practical choice for small n.
  • Numerical stability: It introduces minimal rounding error, which matters for scientific and engineering applications.
  • Ease of understanding: The triple-loop structure maps directly to the mathematical definition, making it the standard teaching example for matrix multiplication.

How Naive Recursive Works

Overview

1 The Idea

This method uses the same divide-and-conquer structure as Strassen: split each matrix into 4 quadrants, recursively multiply sub-matrices, then combine. However, it follows the standard block multiplication formula directly, requiring 8 recursive multiplications per level.

2 The 8 Products

Each result quadrant needs exactly 2 recursive multiplications:

C11 = a·e + b·g
C12 = a·f + b·h
C21 = c·e + d·g
C22 = c·f + d·h

3 Complexity

T(n) = 8·T(n/2) + O(n²) ⇒ O(n3)

With 8 sub-problems (vs Strassen's 7), the exponent remains log28 = 3. This is asymptotically the same as the triple-loop naive method, but uses recursion and more memory.

4 Why Study It?

  • It shows the baseline for divide-and-conquer matrix multiplication.
  • Comparing 8 vs 7 products makes Strassen's improvement concrete and measurable.
  • It demonstrates that recursion alone doesn't improve complexity — the reduction in sub-problems is what matters.

Why 7 Instead of 8?

Naive block multiplication needs 8 sub-matrix multiplications to compute 4 result quadrants (each needs 2). Strassen discovered that by combining quadrants before multiplying, you can compute 7 products whose results, when added/subtracted, yield all 4 result quadrants.

The trick: P1 = (a+d)(e+h) contributes to multiple result quadrants simultaneously. The 7 products share information through algebraic cancellation. The 14 extra additions are O(n²), which is dominated by the O(n2.807) recursive savings for large n.
